# Project Nexus: AI Mastery via Construction

**Project Nexus** is a comprehensive, multi-phase curriculum designed to take engineers from AI fundamentals to advanced agentic orchestration. The goal is to build a **Personal Intelligence Command Center (PICC)**—a local/cloud hybrid environment that automates complex research, coding, and system design tasks to exponentially increase professional productivity.

---

## 🎯 The Mission
To master the art of "Agency" by building it. Project Nexus rejects passive learning in favor of **Constructionist Pedagogy**. Every concept—from tokenization to multi-agent swarms—is learned by implementing functional code on local hardware.

---

## 🏛️ The Curriculum (15-Phase Roadmap)

### Part I: Foundations & Memory
*   **Phase 1: The Foundation** – Mastering inference, context windows, and establishing the local CLI cockpit.
*   **Phase 2: The Librarian (RAG)** – Building persistent semantic memory using Vector Databases (ChromaDB).
*   **Phase 3: The Hand (Agency)** – Implementing tool-use and high-priority system directives.

### Part II: Standardized Integration
*   **Phase 4: The Bridge (MCP)** – Connecting AI to the real world via the Model Context Protocol.
*   **Phase 5: The Rig (Harnessing)** – Designing robust software wrappers for deterministic AI actions.
*   **Phase 6: The Council (Orchestration)** – Managing multi-agent peer-review loops via the **Blackboard Protocol**.

### Part III: Advanced Multimodality & Team Engineering
*   **Phase 7: The Master (Skills)** – Mastering "Skill Engineering" and high-level autonomous terminal agents.
*   **Phase 8: The Voice** – Implementing local, high-speed Speech-to-Text (Whisper) for hands-free coding.
*   **Phase 9: The Forge (Team Tiers)** – Optimizing hardware by routing tasks between VRAM (Fast) and RAM (Deep) tiers.
*   **Phase 10: The Link (Web Portal)** – Deploying mobile-responsive visual dashboards for remote command.

### Part IV: The Infinite Frontier (Phase 11+)
*   Vision-Language Models (VLM), Self-Building Tools, Advanced Agent Steering, and Industrial-Scale MCP Infrastructure.

---

## 🛠️ The Tech Stack
*   **Intelligence:** Local Qwen 3.5 (9B/35B), Phi-4, and Gemma 3 models via **LM Studio**.
*   **Interfaces:** OpenCode (Autonomous Terminal), Gemini CLI (Strategic Mentor), and Open WebUI (Visual Portal).
*   **Protocols:** Model Context Protocol (MCP) and the **Opal Blackboard Protocol** (`OPAL.md`).
*   **Hardware Optimization:** 4-bit KV Cache Quantization, Flash Attention, and tiered memory offloading.

---

## 🛡️ The Defensive Engineering Pillars
The project codifies "Hardened AI" practices to handle the probabilistic nature of LLMs:
1.  **JSON Hygiene:** Robust hunter-parsers to handle model noise.
2.  **State Persistence:** Using neutral Markdown files (`OPAL.md`) as a "Source of Truth" across different AI tools.
3.  **VRAM Budgeting:** Precise compute allocation to ensure real-time performance on consumer GPUs (e.g., RTX 3080 Ti).
4.  **Instruction Anchoring:** Strict system directives to prevent "Persona Drift" during long-running tasks.

## 🛰️ Analogy Framework
While Project Nexus uses **RF Engineering** (Signal Processing, Link Budgets, Modulation) as a consistent analogy for AI concepts, the skills acquired are universally applicable to any high-level software engineering or data science workflow.
