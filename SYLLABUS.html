Master Manifest

Build a hybrid Local/Cloud "Personal Intelligence Command Center" (PICC) that automates RF engineering research and code development. This project serves as a living curriculum for the user to master LLMs, Agentic Workflows, and the Model Context Protocol (MCP).

The Tech Stack

Interfaces: Gemini CLI / OpenCode / Aider
Backend: LM Studio (Local Qwen 35B) / Groq Cloud
Memory: ChromaDB (Vector DB)
Frameworks: PydanticAI / LangChain / MCP

The Defensive Engineering Pillar

  • JSON Hygiene: Build parsers that hunt for { and } to ignore model chatter.
  • The Template Rule: Enforce strict alternating User -> Assistant patterns.
  • The VRAM Ceiling: Prioritize models that fit 100% in VRAM to avoid the 1 t/s bottleneck.
  • Anchoring: Use System Directives to prevent prompt drift in multi-turn tasks.

🛠️ Troubleshooting Guide

🏮 Trap 1: The "Empty Response"

Cause: Context Overflow or Model Refusal. Fix: Reduce max_tokens or lower temperature to 0.1.

🏮 Trap 2: The "400 BadRequest"

Cause: Broken User-Assistant alternation. Fix: Ensure your message list always follows U -> A -> U -> A.

🏮 Trap 5: The "Latency Timeout"

Cause: Local TTFT exceeds 60s. Fix: Set `timeout=300.0` in your OpenAI client config.

Phase 1: The Foundation

Subject: Inference, Tokens, and the Physical Layer of AI

🎯 Learning Objectives

  • Define a "Token" and explain how it relates to RF sampling.
  • Measure model throughput using Tokens Per Second (t/s).
  • Analyze the Latency-Weight Tradeoff between Local and Cloud models.

📖 Technical Deep Dive

Tokens: AI text is sampled into tokens. A 1,000-token prompt is a 1,000-sample buffer. (Analogy: IQ Samples)

Context Window: The observation buffer size. If too small, coherence is lost. (Analogy: FFT Window Size)

Tradeoffs: Local (Privacy/No Cost) vs Cloud (Speed/Massive Wisdom).

Generation Prompt: "Write a Python script to benchmark a local LM Studio inference server. Calculate Tokens Per Second (t/s) by dividing completion_tokens by duration."
