Master Manifest

Build a hybrid Local/Cloud "Personal Intelligence Command Center" (PICC) that automates RF engineering research and code development. This project serves as a living curriculum for the user to master LLMs, Agentic Workflows, and the Model Context Protocol (MCP).

The Tech Stack

Interfaces:

Gemini CLI / OpenCode / Aider

Backend Inference:

LM Studio (Local 35B) / Groq Cloud

Standardized Protocol:

Model Context Protocol (MCP)

Source of Truth:

AGENTS.md (Neutral Blackboard)

The Defensive Engineering Pillar

  • JSON Hygiene: Hunter-parsers for { } ignore conversational noise.
  • The Template Rule: Enforce strict U->A role-playing patterns.
  • The VRAM Ceiling: Prioritize models that fit 100% in VRAM to avoid the 1 t/s bottleneck.
  • Anchoring: Fixed System Directives prevent prompt drift.
  • Portable Memory: Use `AGENTS.md` as the neutral source of truth for cross-agent synchronization.

🛠️ AI Troubleshooting Guide

🏮 Trap 1: The "Empty Response" (Context & Refusal)

Symptom: The AI returns an empty string or just "...". Cause: Context Overflow or Model Refusal. Fix: Reduce max_tokens or simplify the prompt.

🏮 Trap 2: The "400 BadRequest" (Template Collision)

Symptom: Error rendering prompt with jinja template. Cause: Broken User-Assistant alternation. Fix: Ensure message list always follows U -> A -> U -> A.

🏮 Trap 3: The "Runaway Agent" (Over-generation)

Symptom: AI generates 5,000+ tokens. Cause: High temperature or no max_tokens. Fix: Set max_tokens and temp=0.1.

🏮 Trap 5: The "Latency Timeout"

Symptom: `openai.APITimeoutError`. Cause: Local TTFT exceeds 60s for complex prompts. Fix: Set `timeout=300.0` in client initialization.
